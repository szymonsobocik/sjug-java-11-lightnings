JEP 320 - Remove Java EE and CORBA Modules

JEP 320 - Remove Java EE and CORBA Modules

  • Deprecated in JDK 9

Java EE modules

  • JAX-WS (Java API for XML-Based Web Services)
  • JAXB (Java Architecture for XML Binding)
  • JAF (the JavaBeans Activation Framework)
  • Common Annotations

Na początku wersje w Java SE były identyczne z wersjami w Java EE.
Później wersje w Java EE ewoluowały:
- ficzery nie mające znaczenia dla Java SE
- problemy z utrzymaniem
- było możliwe pobranie wersji standalone i nadpisanie, ale żadko to było stosowane

CORBA

  • Dates back to 1998 (Java 1.4)
  • Evolves outside Java Community Process
  • No modern applications with CORBA in Java
dates back to 1998
- żyje osobno

Subsets of Java EE modules

  • JTA (Java Transaction API) - Java 1.3
  • J2EE Activity Service for Extended Transactions - Java 1.5

Modules removed

java.xml.ws (JAX-WS, plus the related technologies SAAJ and Web Services Metadata)
java.xml.bind (JAXB)
java.activation (JAF)
java.xml.ws.annotation (Common Annotations)
java.corba (CORBA)
java.transaction (JTA)
java.se.ee (Aggregator module for the six modules above)
jdk.xml.ws (Tools for JAX-WS)
jdk.xml.bind (Tools for JAXB)

Release Cadence

Before

Plan:

  • release by flagship features
  • major release every 2 years

Reality:

  • Java 7 took 5 years
  • Java 8 took 3 years
  • Java 9 took 3.5 years

Now

  • Fixed six-month release cadence
    March and September
  • Ship everything that is ready
  • All are major releases - no `beta versions`
Wszystko co będzie skończone będzie wypuszczane.
+ completed features get out earlier
